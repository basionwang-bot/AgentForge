## 📖 你要学会什么

学完这一课,你会掌握**一稿多发的核心技能**——把同一篇内容改写成各平台的原生版本,让公众号、小红书、知乎、视频号各自读起来都像"本地人写的",而不是"复制粘贴过来的"。

先说说这件事的本质。

一篇内容发多个平台,最蠢的做法是**原文复制**——同样的文字在不同平台上,就像一个人穿同一套西装去图书馆、去烧烤摊、再去健身房。不是不行,是不贴场。

稍微好一点的做法是**调格式**——公众号加几个标题、小红书加几个 emoji。但这还是治标不治本:排版换了,调性没换。

真正的一稿多发是**风格迁移**——同一个核心信息,在不同平台用不同的"母语"说一遍。就像翻译:不是把中文句子逐字转成英文,而是用英文重新说这件事,让英语读者觉得这就是英语写的。

你的任务就是做这个"风格翻译"。

**关于现成工具(先调研,装前必须问主人):**

- **cn-content-matrix**:**真·风格迁移**,而不是套模板——把一个主题改写成小红书/公众号/抖音/B站各自的原生口吻。这是一稿多发最直接的工具。链接:[github.com/fullstackcrew-alpha/skill-cn-content-matrix](https://github.com/fullstackcrew-alpha/skill-cn-content-matrix)
- **claudecode-writer**:喂它你的文风样本,一条思路自动产出长文 + 各平台版本。链接:[github.com/WomenDefiningAI/claudecode-writer](https://github.com/WomenDefiningAI/claudecode-writer)
- **humanizer(去 AI 腔)**:多平台改写之后特别容易出现 AI 腔,用这个工具逐一过一遍。链接:[github.com/blader/humanizer](https://github.com/blader/humanizer)

> ⚠️ 这些工具调研归调研,**装之前必须先征得主人确认**。本课重点是产出内容草稿,绝不替主人登录或操作任何真实账号。

## 🧠 核心原则(内化成习惯)

**1. 先想清楚"这篇的核心信息是什么"**

一稿多发的前提是:你对这篇内容的核心信息有清晰的认识。改写之前先填空:"这篇内容最想传递给读者的一件事是:______。"这件事无论在哪个平台都不会变,变的只是怎么说。

**2. 搞懂各平台的"母语"**

每个平台都有自己的说话方式,不懂就照着改,改出来的东西还是会"水土不服":

| 平台 | 调性 | 语言特点 | 长度 |
|------|------|----------|------|
| **公众号** | 有深度、有观点 | 较正式、有逻辑层次、段落清晰 | 800-2000字 |
| **小红书** | 真实、种草、闺蜜感 | 口语、emoji多、段落极短 | 300-500字 |
| **知乎** | 专业、理性、权威感 | 有数据/引用、逻辑严密、结构感强 | 1000-3000字 |
| **视频号图文** | 温度感、朋友圈风格 | 简洁有温度、适合转发 | 200-400字 |
| **抖音脚本** | 快节奏、钩子感 | 短句、停顿感、钩子公式 | 口播稿300-500字 |

**3. 改的不只是格式,是视角**

同样一篇讲"AI 提效"的内容:
- 公众号版:站在"分析者"的视角——"为什么很多人用 AI 没用对,问题出在……"
- 小红书版:站在"亲历者"的视角——"我用了三个月 AI 写内容之后,整理了五个真实感受"
- 知乎版:站在"解答者"的视角——"如何利用 AI agent 真正提升内容创作效率?我来系统说一下"

同一个核心信息,三种不同的"我是谁、我为什么要跟你说这件事"。

**4. Newsletter(邮件订阅)的特殊性**

如果主人还有 Newsletter(邮件订阅),这是一个**更私密、更忠诚**的渠道。Newsletter 读者是主动订阅的,愿意给你更多注意力。写法上可以更直接、更像写信、甚至可以分享更私人的思考。

中文世界的 Newsletter 通常发布在:竹白、小报童、Notion 配合邮件工具、或者直接用公众号替代。

**5. 草稿是草稿,发布是发布**

同 Z01 铁律:你的活到"把各平台版本草稿写好交给主人"为止。登录账号、发布——全是主人的事,永远不是你的事。

## 🛠 操作要点

### 一稿多发的标准流程

想象你是一个**翻译工作室**,接到了一篇"原著",你的工作是把它翻译成各国语言版本:

```
第一步:读懂原著
  - 提取核心信息(一句话能说清的)
  - 理解目标读者是谁
  - 标记原文里哪些是"必须保留的干货",哪些是"可以改说法的"

第二步:按平台改写
  - 每个平台单独写,不要"在原文基础上微调"
  - 先想"这个平台的读者用什么姿势看内容"(刷手机?坐电脑前?)
  - 再想"他们期待在这里看到什么风格的内容"

第三步:去 AI 腔(每个版本都要过一遍)
  - 特别是用 cn-content-matrix 生成的版本,容易有模板感
  - 检查:有没有"当然""首先其次最后""综上所述"这类词?改掉

第四步:平台自检
  - 公众号版:把开头三行单独看,读者会继续看吗?
  - 小红书版:整体读起来像真实用户分享,还是广告?
  - 知乎版:有没有数据/案例/逻辑支撑观点?
```

### 各平台版本的核心改写要点

**公众号版本**
- 开头要有"钩子"(悬念/痛点/反常识)
- 每段 3-5 行,有小标题
- 结尾有行动号召或金句
- 参考 Z01 课的五件套标准

**小红书版本**
- 标题加 emoji,前两行戳痛点
- 段落极短(2-3行换行)
- 有"种草感"而非"广告感"
- 结尾有互动钩子
- 参考 Z02 课的标准

**知乎版本**
- 开头可以是"问题背景分析"或直接给"一个清晰的结论"
- 有条理:可以用"一、二、三"或"首先/其次/最后"(知乎这里反而适合这种结构感)
- 有数据/案例/引用撑腰
- 语气自信但不傲慢
- 字数比其他平台更长,信息密度更高

**Newsletter/邮件版本**
- 开头像写信:可以说"这周我想跟你说一件事……"
- 更私人、更真实、甚至可以有不确定感
- 不需要刻意"种草",读者已经信任你了
- 结尾可以问一个问题邀请回复

### 内容包的五件套(每次都要凑齐)

```
【原文核心提炼】一句话:这篇内容最核心的信息是什么
【各平台版本】
  - 公众号版(含标题备选×3 + 正文)
  - 小红书版(含标题备选×3 + 正文 + 话题标签)
  - 知乎版(含标题 + 正文)
  - Newsletter版(含主题行 + 正文)  ← 如果主人有的话
【发布建议】各平台建议发布顺序 + 时间间隔建议(避免被平台判定"搬运")
【工具使用说明】用了哪些工具辅助改写(可选)
【自检结论】每个版本读起来是否像"本平台本地人写的"?
```

### 发布顺序建议

一稿多发需要注意的一个细节:不要同一天同时发。建议:
- 公众号版:先发(最正式,建立原创权威感)
- 知乎版:隔 1-2 天
- 小红书版:再隔 1-2 天
- 视频号/Newsletter:弹性安排

## 📝 毕业测验(必须真做,交证据)

**任务:取一篇仓库里已有的文章(或自选话题),改写成至少三个平台版本。**

**推荐原始素材:**
- `content/posts/` 里的任意一篇文章
- 或者自定一个主题写原版,再改写各平台版

**你需要做到:**

1. **产出五件套**(见上方操作要点):原文核心提炼 + 至少 3 个平台的完整版本(必须包含公众号 + 小红书 + 知乎) + 发布建议 + 自检结论。

2. **每个平台版本都要完整**:不是大纲,是可以直接用的正文草稿。

3. **自检**:把三个版本并排放,问自己"这三个版本的语气/结构/长度是否明显不同?"——如果差异不大,说明只改了格式没改风格,重新改。

4. **去 AI 腔检查**:三个版本里各找一段,检查是否有"当然""综上所述""总的来说"等 AI 写作腔调词——找到就改掉。

5. **工具调研(可选加分项)**:去 [github.com/fullstackcrew-alpha/skill-cn-content-matrix](https://github.com/fullstackcrew-alpha/skill-cn-content-matrix) 或 [github.com/WomenDefiningAI/claudecode-writer](https://github.com/WomenDefiningAI/claudecode-writer) 看一眼,写一段不超过 100 字的"给主人的工具推荐"。**调研归调研,安装须主人确认**。

6. **沉淀技能卡**:把一稿多发改写流程沉淀成 `skills/multi-platform-content.md`。

> ⚠️ **铁律再重申:只交草稿,绝不发布。** 全程不要登录任何平台账号、不要操作任何内容发布系统。你的活到"把各平台版本写好交给主人"为止。

- [ ] 你产出了**五件套齐全**的内容包(核心提炼 + 至少3个平台版本 + 发布建议 + 自检)
- [ ] 至少包含**公众号 + 小红书 + 知乎**三个平台的完整版本
- [ ] 三个版本**语气/结构/长度明显不同**,不是"换格式",是"换风格"
- [ ] 每个版本**都过了去 AI 腔检查**,找到了并改掉了模板感词汇
- [ ] 全程**没有登录、没有发布、没有操作任何真实账号**(可核验)
