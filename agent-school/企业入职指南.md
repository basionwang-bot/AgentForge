# 🏢 企业入职指南 · 为你的内部 agent 办入职(占位)

> 占位文档(一页骨架)。说明企业 / 团队如何基于 AgentForge 这套体系,
> 为自己的内部 agent 定制一条"入职"路径——让 AI 真正学会**你们团队的活**,而不只是通用技能。

---

## 思路:同一套机制,换成你们的料

AgentForge 的内核是一套**可复用的"入职协议"**:面谈诊断 → 任务驱动选课 → 真执行 + 独立考官 → 技能安家。
企业要做的,只是把"料"换成自己的:

| 组件 | 公开版 | 企业定制版 |
|------|--------|-----------|
| **流程课** | 通用做事纪律(J01–J07) | 你们的**内部流程**(审批/发布/合规步骤) |
| **工具课** | 通用工具(pandas/ffmpeg…) | 你们的**内部工具 / 系统 / API** |
| **考官协议** | 独立考官 + 实证毕业 | **同一套**,不打折 |
| **课程位置** | `courses/`(公开) | [`courses/private/`](courses/private/)(私有,不公开) |

## 三步落地

1. **写私有课**:把内部流程 / 工具,按 [`出课标准.md`](出课标准.md) 写成课,放进 `courses/private/`。
2. **定制入职指令**:复制 [`onboard.md`](onboard.md),把选课范围指向 `courses/private/`,面谈问题换成贴合团队的。
3. **统一安家**:学完的技能卡按 [`毕业安家.md`](毕业安家.md) 装到内部 agent 的全局技能目录,跨会话生效。

## 为什么值得

通用 AI 助手不懂你们的内部流程;招人培训又慢又贵。**给 AI 办一次入职**,它就带着你们团队的 know-how 上岗——
而且经过独立考官把关,本事是**验过的**。
