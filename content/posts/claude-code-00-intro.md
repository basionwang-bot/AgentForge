## 先说人话：这门课到底教什么？

你大概率玩过 AI 聊天（DeepSeek、豆包、ChatGPT）。那种感觉是这样的：**你问一句，它答一句；你复制、粘贴，然后还是自己埋头干。** 它像隔着玻璃跟你比划——能出主意，但碰不到你的活儿。

Claude Code 不是这么个东西。想象一下，你身边突然多了一个**新来的实习生**：脑子灵、手脚快，工具箱还塞得满满当当。你不用什么都嚼碎了喂到他嘴边——

- 他能**自己翻你电脑里的文件**（不是你贴给他看，是他自己跑去找）
- 他能**自己挽起袖子上手**（装东西、跑测试、把活儿交上去）
- 他能**自己改、自己检查、不行再改一遍**
- 你可以站旁边盯着他干，随时喊停纠正；也可以**转身走开**，回来直接收成果

说白了：**聊天 AI 是个隔着屏幕给你支招的"顾问"；Claude Code 是个能下场干的"实习生"。** 一个动嘴，一个动手。

这门课从头到尾就教一件事：怎么把这个实习生带好——带到你一个人，能干出一支小团队的活。

> ⚠️ 提醒一下：这门课假设你已经会在终端里敲 `claude` 把它叫醒。要是你连"终端"是个啥都还没概念，建议先看小屋里那篇《AI 工具总览》，从 Cursor 这类能点点点的工具入门，再回来学这门课。

## 整门课就靠这一句话撑着（请刻进脑子）

后面所有的招数，其实都是从这一句话里长出来的：

> **🧠 Claude 干活时，靠的是一张大小有限的"写字台"。台子堆得越满，它越笨。**

什么意思？

想象一下，Claude 干活的时候，面前有一张**写字台**。你说的每句话、它读过的每个文件、它跑命令蹦出来的每一行结果，**全都摊在这张桌子上**，一张纸一张纸地占着地方。

可桌子就那么大。堆太满了会怎样？跟人一样：

- 它开始**忘事**——前面刚说好的规矩，回头就不认了
- 它开始**抓不住重点**——被一堆没用的纸片埋住了，捞不着干货
- 它开始**犯傻**——明明文件里白纸黑字写着，它楞是没看见

所以你记住：**用得好不好，差别根本不在"会不会写漂亮的指令"，而在"会不会帮它收拾这张桌子"。**

几乎所有招数，归根到底都在回答三个问题：

1. 怎么让它**第一次就听懂**？（→ 模块二：精准提问）
2. 怎么让它**自己检查自己**？（→ 模块四：验证闭环）
3. 怎么帮它**把桌子收拾干净**？（→ 模块五：上下文管理）

这三件事都顺了，你才有资格进第四阶段：**自动化与规模化**——让好几个实习生同时替你干活（模块七）。

## 学习路线图：你该按什么顺序学？

这门课一共八个模块，但**别想着一口气全啃完**。照下面这张表对号入座就行：

| 你是谁 | 推荐路线 | 学完能干嘛 |
|--------|----------|------------|
| **完全新手 / 不懂技术 / 老师、独立创业者** | 模块 ① ② ③ ⑧ | 能稳稳地让它把一件件具体的事办妥，不返工、不翻车 |
| **想认真提效的个人** | 加学 ④ ⑤ ⑥ | 能让它自己检查、自己收拾桌子，还能配好你专属的干活环境 |
| **想"一个人开一支队伍"的狠人** | 再加 ⑦ | 能让它在你睡觉时自动干活、批量处理、好几摊一起开工 |

**建议所有人都先把 ①②③⑧ 这四块吃透。** 这是地基。地基不稳，后面学再多花活，都是空中楼阁。

## 这门课的每一篇长什么样

为了让新手真看得懂，我把官方那份"写给专家看"、密密麻麻的大纲，**一条条翻成了大白话**，每篇都按这个套路展开：

- **这是什么 / 为什么重要**（先把"为啥要学"讲明白，不让你死记硬背）
- **怎么做**（一步一步，能照着抄）
- **现场例子**（真实场景，不拿玩具糊弄你）
- **新手最容易踩的坑**（提前帮你拉根绳子绊一下，让你早点摔在软垫上）

## 八个模块速览（点进去看详细版）

1. **模块一·换脑子** —— 从"对它下命令"变成"带一个干活的人"。心态摆正，是一切的前提。
2. **模块二·提问的精度** —— 话说得越准，返工越少。四个改写模板 + 让它反过来采访你的杀手锏。
3. **模块三·黄金工作流** —— 探索 → 计划 → 实现 → 提交。官方推荐的标准四步，以及"什么时候别用它"。
4. **模块四·验证闭环** —— 让它自己检查自己。这是"打杂的"和"靠谱实习生"的分水岭。
5. **模块五·上下文管理** —— 收拾那张"桌子"的全套手法。高手和普通人的真正分界线。
6. **模块六·环境配置** —— 配一次，省一年。CLAUDE.md、权限、Skills、Hooks、MCP。
7. **模块七·自动化** —— 让它在你不在场时也照样干活。定时任务、批量铺开、好几摊并行。
8. **模块八·五大翻车现场** —— 官方总结的五种常见失败，每个配"症状 + 解药"。最适合新手提前看。

最后还有一篇 **《附录·命令速查表 + 官方资料》**，建议收藏，随时翻。

## 给新手的三句心里话

1. **别指望"一句话让它干完一切"。** 它的本事是"被你带得好"，不是"会读心术"。你越会拆活、越会把标准说清楚，它越神。
2. **翻车很正常，关键是会复盘。** 模块八那五个坑，你大概率每个都得亲自踩一遍。踩了别慌，回来翻解药。
3. **从今天起，把它当"同事"而不是"工具"。** 你怎么跟一个靠谱的新同事交代活，就怎么跟它说话。这个弯一转过来，这门课就成功了一半。
